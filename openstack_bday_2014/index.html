OpenStack 4th Birthday

TripleO

chicken/egg done right

The project

A program aimed at installing, upgrading and operating OpenStack clouds using OpenStack's own cloud facilities as the foundations.

Automate fleet management at datacentre scale and yet scale down to as few as 2 machines

Raw but usable today

How it works

Undercloud and Overcloud (and Seed?)

All core components involved with Nova Baremetal / Ironic and Heat working harder

Worth a thousand words

The spicy part

A collection of tools used for imaging, Heat templates management and nodes config management

Plus Tuskar/UI

... some living their own lives, like DIB and and the OS-* tooling

diskimage-builder

(and tripleo-image-elements)
  • Consumes upstream cloud images
  • Extracted to local filesystem
  • Software installed (and configured) via "elements"

tripleo-heat-templates

(and merge.py)
  • Undercloud and Overcloud provisioning
  • Heat templates on steroids [1]

1. http://lists.openstack.org/pipermail/openstack-dev/2014-April/031915.html

os-*-config

(collect, refresh, apply)
  • Collect metadata and trigger hook on changes
  • Run hook's states
  • Apply metadata to config files via Mustache templating

Tuskar

Deployment control via REST API

Also provides a web UI but for now has limited capabilities

CI

Limited in scope

Runs on Ubuntu and Fedora

The goods

  • Try out OpenStack from master
  • Or take it easy with RDO's instack

The bads

  • Supports only some topologies
  • Yet evolves pretty quickly
